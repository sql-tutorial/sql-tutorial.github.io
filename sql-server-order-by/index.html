Введение в предложение ORDER BY SQL Server

Предложение ORDER BY является опциональным предложением оператора SELECT. Предложение ORDER BY позволяет сортировать набор результатов запроса по одному или нескольким столбцам.

Вот синтаксис предложения ORDER BY: 

SELECT
    select_list
FROM
    table_name
ORDER BY 
    column_name | expression [ASC | DESC ];

В этом синтаксисе:

Сначала укажите имя столбца или выражение, которое вы хотите отсортировать, в предложении ORDER BY.

Если вы используете несколько столбцов, предложение ORDER BY сначала отсортирует строки по первому столбцу, затем отсортирует отсортированные строки по второму столбцу и т. д.

Столбцы в предложении ORDER BY должны соответствовать либо столбцу в списке выбора, либо столбцам, определенным в таблице, указанной в предложении FROM.

Во-вторых, укажите порядок сортировки, используя ключевое слово ASC или DESC.

Ключевое слово ASC сортирует строки от меньших к большим, а ключевое слово DESC сортирует строки от больших к меньшим.

Оба варианта ASC и DESC являются необязательными. Если вы явно не укажете ASC или DESC, предложение ORDER BY по умолчанию будет ASC.

Кроме того, SQL Server рассматривает NULL как наименьшее значение. Это означает, что он поместит NULL перед другими значениями после сортировки.

При обработке оператора SELECT, содержащего предложение ORDER BY, SQL Server обрабатывает предложение ORDER BY последним.

Пример предложения ORDER BY для SQL Server

Мы будем использовать таблицу клиентов из примера базы данных из демонстрации.

1) Сортировать набор результатов по одному столбцу в порядке возрастания.

В следующем операторе используется предложение ORDER BY для сортировки клиентов по их именам в порядке возрастания: 

SELECT
    first_name,
    last_name
FROM
    sales.customers
ORDER BY
    first_name;

Выход:

В этом примере мы не указываем ASC или DESC, предложение ORDER BY по умолчанию равно ASC. Поэтому приведенный выше запрос эквивалентен следующему запросу: 

SELECT
    first_name,
    last_name
FROM
    sales.customers
ORDER BY
    first_name ASC;

2) Сортировать набор результатов по одному столбцу в порядке убывания.

В следующем операторе используется предложение ORDER BY для сортировки клиентов по их именам в порядке убывания: 

SELECT
    firstname,
    lastname
FROM
    sales.customers
ORDER BY
    first_name DESC;

Выход:

В этом примере мы явно используем параметр DESC, поэтому предложение ORDER BY сортирует строки по значениям в столбце first_name в порядке убывания.

3) Сортировка набора результатов по нескольким столбцам

В следующем операторе используется предложение ORDER BY для сортировки клиентов сначала по городам, а затем по именам: 

SELECT
    city,
    first_name,
    last_name
FROM
    sales.customers
ORDER BY
    city,
    first_name;

Выход:

4) Сортировка набора результатов по нескольким столбцам в разном порядке.

В следующем операторе используется предложение ORDER BY для сортировки клиентов по городам в порядке убывания, а затем сортировки клиентов по их именам в алфавитном порядке: 

SELECT
    city,
    first_name,
    last_name
FROM
    sales.customers
ORDER BY
    city DESC,
    first_name ASC;

Выход:

5) Сортировать набор результатов по столбцу, которого нет в списке выбора

SQL Server позволяет сортировать набор результатов по столбцам, указанным в таблице, даже если эти столбцы не отображаются в списке выбора.

Например, в следующем операторе используется предложение ORDER BY для сортировки клиентов по штатам, даже если столбец штата не отображается в списке выбора: 

SELECT
    city,
    first_name,
    last_name
FROM
    sales.customers
ORDER BY
    state;

Выход:

Обратите внимание, что столбец state определен в таблице customers. Если это не так, то у вас будет недействительный запрос.

6) Сортировка результирующего набора по выражению

Функция LEN() возвращает количество символов в строке.

В следующем операторе функция LEN() используется в предложении ORDER BY для сортировки клиентов по длине их имен: 

SELECT
    first_name,
    last_name
FROM
    sales.customers
ORDER BY
    LEN(first_name) DESC;

Выход:

7) Сортировка по порядковому положению столбцов

SQL Server позволяет сортировать результирующий набор на основе порядковых номеров столбцов, отображаемых в списке выбора.

Следующий оператор сортирует клиентов по имени и фамилии. Но вместо явного указания имен столбцов он использует порядковые позиции столбцов: 

SELECT
    first_name,
    last_name
FROM
    sales.customers
ORDER BY
    1,
    2;

В этом примере числа 1 и 2, которые появляются после предложения ORDER BY, обозначают столбцы first_name и last_name соответственно.

Использование порядковых номеров столбцов в предложении ORDER BY не рекомендуется по нескольким причинам:

  • Во-первых, столбцы в таблице не имеют порядковых номеров и должны обозначаться именами.
  • Во-вторых, если вы измените список выбора, вы можете забыть обновить предложение ORDER BY соответствующим образом.

Поэтому лучше всего всегда явно указывать имена столбцов в предложении ORDER BY.

Краткое содержание

  • Используйте предложение ORDER BY для сортировки набора результатов по столбцам в порядке возрастания или убывания.
  • Используйте ключевое слово ASC для сортировки строк в порядке возрастания.
  • Используйте ключевое слово DESC для сортировки строк в порядке убывания.
