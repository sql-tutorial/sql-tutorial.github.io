Резервное копирование журнала транзакций SQL Server

Введение в резервное копирование журнала транзакций SQL Server

Если модель восстановления базы данных — FULL или BULK_LOGGED, вы можете создать резервную копию журнала транзакций базы данных.

Перед созданием резервной копии журнала транзакций необходимо создать как минимум одну полную резервную копию. После этого можно создавать любое количество резервных копий журнала транзакций.

Рекомендуется чаще создавать резервные копии журнала транзакций, чтобы:

  • Минимизируйте потерю данных
  • Усечение файлов журнала

Обычно вы создаете полную резервную копию время от времени, например, еженедельно, и создаете серию дифференциальных резервных копий с более коротким интервалом, например, ежедневно. Независимо от резервных копий базы данных, вы создаете журнал транзакций с более частыми интервалами, например, каждый час.

На следующем рисунке показано резервное копирование журнала транзакций:

На этой картинке мы имеем:

  • Две полные резервные копии
  • Три резервных копии журнала транзакций

Первая полная резервная копия содержит идентификатор 1, а вторая полная резервная копия содержит идентификаторы 1, 2 и 3.

Первая резервная копия журнала транзакций содержит идентификатор 2, вторая резервная копия журнала транзакций содержит идентификатор 3, а третья резервная копия журнала транзакций содержит идентификатор 3.

Обратите внимание, что резервные копии журнала транзакций не содержат дублирующихся данных, как полная резервная копия или дифференциальная резервная копия.

Создайте резервную копию журнала транзакций с помощью T-SQL

Для создания резервной копии журнала транзакций используйте оператор BACKUP LOG: 

BACKUP LOG database_name
TO DISK = path_to_backup_file
WITH options;

В этом заявлении:

  • Сначала укажите имя базы данных для резервного копирования журнала транзакций. База данных должна существовать и находиться в состоянии онлайн.
  • Во-вторых, укажите путь к файлу журнала резервного копирования. Путь должен существовать в файловой системе.
  • В-третьих, укажите дополнительные параметры резервного копирования в предложении WITH.

Пример резервного копирования журнала транзакций SQL Server

Давайте рассмотрим пример создания резервных копий журнала транзакций и восстановления базы данных из них.

1) Создание примера резервной копии журнала транзакций

Сначала удалите базу данных HR: 

USE master;
DROP DATABASE IF EXISTS HR;

Во-вторых, создайте новую базу данных HR: 

CREATE DATABASE HR;
GO

В-третьих, убедитесь, что база данных HR находится в режиме ПОЛНОГО восстановления, чтобы выполнить резервное копирование журнала транзакций: 

ALTER DATABASE HR 
SET RECOVERY FULL;

В-четвертых, создайте таблицу People и вставьте строку: 

USE HR;
GO
CREATE TABLE People(
  Id int IDENTITY PRIMARY KEY,
  FirstName varchar(50) NOT NULL,
  LastName varchar(50) NOT NULL
);
INSERT INTO People(FirstName, LastName)
VALUES('John', 'Doe');

В-пятых, создайте полную резервную копию: 

BACKUP DATABASE HR 
TO  DISK = 'D:\backup\hr.bak'
WITH INIT,
NAME = 'HR-Full Database Backup';

В-шестых, вставьте еще одну строку в таблицу People: 

INSERT INTO People(FirstName, LastName)
VALUES('Jane', 'Doe');

В-седьмых, создайте первую резервную копию журнала транзакций: 

BACKUP LOG HR
TO  DISK = N'D:\backup\hr.bak' 
WITH NAME = N'HR-Transaction Log Backup';

Восьмое, вставьте еще одну строку в таблицу People: 

INSERT INTO People(FirstName, LastName)
VALUES('Upton', 'Luis');

Девятое, создайте вторую резервную копию журнала транзакций: 

BACKUP LOG HR
TO  DISK = N'D:\backup\hr.bak' 
WITH NAME = N'HR-Transaction Log Backup';

В-десятых, создайте вторую полную резервную копию: 

BACKUP DATABASE HR 
TO  DISK = 'D:\backup\hr.bak'
WITH NOINIT,
NAME = 'HR-Full Database Backup';

Одиннадцатое, вставьте еще одну строку в таблицу People: 

INSERT INTO People(FirstName, LastName)
VALUES('Dach', 'Keon');

Наконец, просмотрите файл резервной копии: 

RESTORE HEADERONLY   
FROM DISK = N'D:\backup\hr.bak';

2) Восстановить базу данных из резервной копии журнала транзакций

Чтобы восстановить базу данных, можно восстановить вторую полную резервную копию(позиция 4) и последнюю резервную копию журнала транзакций(позиция 5).

Сначала удалите базу данных HR: 

USE master;
DROP DATABASE HR;

Во-вторых, восстановите базу данных из второй полной резервной копии: 

RESTORE DATABASE HR
FROM  DISK = N'D:\backup\hr.bak'
WITH FILE = 4, NORECOVERY;

В этом операторе число 4 указывает на вторую полную резервную копию в файле резервной копии. NORECOVERY переводит базу данных в состояние восстановления, чтобы можно было восстановить резервную копию журнала транзакций.

В-третьих, восстановите резервную копию журнала транзакций с помощью оператора RESTORE LOG: 

RESTORE LOG HR
FROM DISK = N'D:\backup\hr.bak'
WITH FILE = 5, RECOVERY;

В этом операторе число 5 указывает на последнюю резервную копию журнала транзакций. RECOVERY указывает на то, что больше нет резервной копии для восстановления. Таким образом, база данных доступна после завершения восстановления.

Наконец, переключите текущую базу данных на HR и выберите данные из таблицы People: 

USE HR;
SELECT * FROM people;

Выход:

Таблица People содержит четыре строки, указывающие на то, что вы успешно восстановили базу данных из полной резервной копии и резервной копии журнала транзакций.

Краткое содержание

  • Для резервного копирования журнала транзакций модель восстановления базы данных должна быть FULL или BULK_LOGGED.
  • Резервная копия журнала транзакций содержит журнал транзакций базы данных.
  • Используйте оператор BACKUP LOG для резервного копирования журналов транзакций.
  • Используйте RESTORE LOG для восстановления базы данных из резервных копий журнала транзакций.
